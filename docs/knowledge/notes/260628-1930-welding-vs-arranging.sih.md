# 焊接 vs 编排：自反性治理体系的诊断协议

> 从维特根斯坦《哲学研究》序言中提取的元层级问诊框架，经司衡自反性诊断验证有效。

## 一、协议概述

自反性治理体系的核心张力：体系化是必要的（否则无法规模化），但体系化有自我扩张的本能。当体系声称某事物"应该如此"时，读者需要区分两种不同的体系化方式：

- **焊接**（Welding）：强迫思想或被治理对象进入单向线性秩序，不允逆向流动、跨层引用、例外存在。
- **编排**（Arranging）：保持结构化但允许交叉连接、逆向流动和方向修正，结构是工具而非强制。

## 二、诊断四项标准

| 标准 | 焊接的特征 | 编排的特征 |
| ---- | ---- | ---- |
| **可逆性** | 只允许顺向流动（顶层→底层） | 允许逆向、跨层、循环引用 |
| **可剔除性** | 旧内容被覆盖或遗忘 | 旧内容被显式剔除但保留痕迹（archive） |
| **盲区标注** | 声称完备，无盲区声明 | 标注"该系统不知道什么"（@blind_spots） |
| **外部批评构成性** | 批评被视为需要防御的攻击 | 批评被纳入体系结构（review results 存档） |

**如何使用**：当治理体系出现系统性信号时（规则持续增长、Revert 率上升、新成员学习成本增加），应用此协议做一次自反性诊断。四项标准中任意两项偏离编排方向，即表明体系正在从编排滑向焊接，需要触发知止声明的重审。

## 三、诊断实例：司衡自身

司衡在 2026-06-28 自反性诊断中应用此协议，结果为"处于编排与焊接之间"。四项诊断中两项已满足标准（可剔除性：archive/ 存在；外部批评构成性：review-results/ 完整），两项未完全满足（可逆性：ARG-04 证明了逆向流动存在但未制度化；盲区标注：@blind_spots 当时不存在）。

闭合方式：
- 盲区标准：通过 @blind_spots frontmatter 字段标准化满足
- 可逆性标准：通过行迹机制（链 15）的建立满足——意图恢复允许从实现层回推到意图层

## 四、协议的自身约束

此协议本身是编排性的分析工具，不是焊接性的规则系统。它不自称完备，不自称普遍适用，其有效性有待多次诊断实践的验证。当前标注为 `empirical-hypothesis`。

## 五、起源

此协议源自维特根斯坦《哲学研究》序言的阅读（2026-06-28 对话），经司衡自身诊断验证后提取为可复用的分析框架。
