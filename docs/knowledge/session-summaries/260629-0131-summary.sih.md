## 治理决策

1. **孤儿判据**：按文件存在性而非 stage/nature 判定，确保 archive 子树的合法废弃文档不被误伤。
2. **DB 不存 file_path 列**：维持现状，存量不回填，新比对通过 parse 磁盘文件完成。
3. **MVP 范围**：仅 documents 表；metrics 表（IndexCompleted / ValidationCompleted 事件中的 ghost doc_id）单独成案。
4. **安全前置**：先 --prune-dry-run 输出待删清单给用户 review，再 --prune-confirm 真正删除；不默认开启 prune。
5. **三不删规则**：文件存在但 stage X 保留；文件存在但 nature unknown 保留；文件不存在才删。
