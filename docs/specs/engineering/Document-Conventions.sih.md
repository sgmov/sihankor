# 司衡文档规范

> 术层文档。定义司衡治理文档的 frontmatter 字段、认识论标签、外部锚定声明、术语血统引用、目录结构、文件命名与字符约束。法层原则见[《司衡哲学总纲》](../philosophy/SiHankor-Philosophy.sih.md)。

## 一、frontmatter 字段规范

frontmatter 为 YAML，包裹于 `---` 分隔符之间。分隔符是全文唯一允许的水平线。`type` 字段已废除，文档身份由所在目录唯一确定，引擎从路径第一层推断。

### 1.1 字段定义

| 字段 | 格式 | 说明 |
| ---- | ---- | ---- |
| `id` | `YYMMDD-HHMM[-NNN]-语义短名` | 全部必填。文档唯一标识，一经分配永久不变 |
| `stage` | `1/3` `2/3` `3/3` `0/<后继id>` `X` | 全部必填。治理可信度或生命周期状态 |
| `upstream` | 文档 id | spec/proposal/decision/reference 必填，note 可选。治理授权源头，根文档指向自身 |
| `decided-by` | 人名或 `ai-assist` | 仅 decision。决策的意图源头，`ai-auto` 不得用于人需决策的 ADR |
| `verified` | `YYMMDD` | 仅 note。洞察最近确认有效的日期 |

### 1.2 stage 编码语义

stage 编码形式统一，语义由文档所在目录（nature）决定：

| nature | stage 推进语义 |
| ---- | ---- |
| spec/proposal/decision/reference | 1/3 起草 -> 2/3 审查 -> 3/3 定稿。回答文档可被下游信任的程度 |
| note | 1/3 初现 -> 2/3 人类确认 -> 3/3 已充分验证。回答洞察成熟度 |

`0/` 编码后继 id，读者看到 stage 即知意图去了哪里。`X` 表示废弃。

## 二、认识论标签使用规则

体系内每条原则、指南和主张必须携带以下标签之一，标注其认识论地位。一条主张可携带多个标签。不允许无标签的主张进入原则层。标签决定主张的权威来源与可修正门槛。

### 2.1 external-anchor

主张锚定到体系之外已被确立的科学定律或外部成果时使用。其权威来自体系之外。

格式要求：声明锚定对象（外部定律名称）与对应关系（外部定律如何与主张描述同一现象）。声明格式见第三节。

### 2.2 empirical-hypothesis

主张为可证伪的经验假设时使用。其权威来自经验检验。

必须包含可证伪条件：明确陈述在何种可观测情形下该主张被证伪。缺失可证伪条件的经验假设不成立。

### 2.3 tautology

主张为逻辑必然，由定义或因果方向决定，无需经验检验时使用。无需外部锚定，其权威来自逻辑自洽。如"代码按定义由有意图的主体产出"。

### 2.4 design-corollary

主张从其他已确立原则推导而来时使用。

必须标注推导自哪条原则：声明推导源头（如"推导自道二"）。无推导源头的 corollary 不成立。

### 2.5 constructed-framework

主张为在经验约束下被建构的有效框架，其有效性有待外部验证时使用。

必须声明当前验证状态：显式标注当前处于"内部检验已通过、外部独立验证未完成"等状态。不得声称已获外部验证。

## 三、外部锚定声明格式

外部锚定声明紧随主张陈述，不超过 3 行。三行分别承载主张与标签、锚定对象、对应关系。

```text
{原则名}：{主张陈述}。标签：{标签列表}。
锚定：{外部科学定律名称}。
对应：{外部定律如何与主张描述同一现象}。
```

示例（道一）：

```text
道一：发散自-然，收敛必-为。标签：empirical-hypothesis + external-anchor。
锚定：热力学第二定律。
对应：孤立系统熵增即发散自-然，局部熵减需要做功即收敛必-为。
```

东方哲学术语与西方科学术语描述同一现象时互为锚定，不存在谁更"标准"的问题。

## 四、术语血统引用规范

引用司衡核心术语时，须追溯其道家血统，声明源出、原义、司衡取义与关系。

### 4.1 引用要素

每条术语血统引用包含四要素：

| 要素 | 说明 |
| ---- | ---- |
| 源出 | 典籍名称与章次，如《道德经》第25章 |
| 原义 | 典籍中的原始含义 |
| 司衡取义 | 司衡体系中赋予的含义 |
| 关系 | 与原义关系：保留、延伸或限定 |

### 4.2 引用格式

```text
{术语}：源出{典籍}{章次}。原义为{原义}。司衡取"{取义}"义。与原义关系：{保留/延伸/限定}。
```

详细考证见[《司衡术语血统表》](../philosophy/SiHankor-Terminology-Lineage.sih.md)。司衡七个核心术语均为道家血统：五个直接引《道德经》《庄子》原文，两个为道家义理延伸概念。无一儒法家源头，无一自造。

## 五、目录结构与文件命名

### 5.1 目录结构

文档身份由所在目录唯一确定，引擎从路径第一层推断 nature：

| 目录 | nature | 用途 |
| ---- | ---- | ---- |
| `specs/` | spec | 系统定义：系统是什么 |
| `proposals/` | proposal | 变更提议：我们提议改变什么 |
| `decisions/` | decision | 决策记录：为什么这样选 |
| `reference/` | reference | 参照标准：术语指什么 |
| `knowledge/notes/` | note | 实践洞察：我们学到了什么 |
| `knowledge/drafts/` | 无 | 构思碎片，非 .sih.md，无 frontmatter |
| `archive/` | 无 | 废弃文档，stage 为 X 或 0 |

所有目录允许子目录，最多三层。拆分阈值：单目录文件数大于 30。

### 5.2 文件命名

```text
文件名 = 语义词-语义词-... .sih.md
```

每个语义词首字母大写（PascalCase），连字符连接。`SiHankor` 保留大写 S 与大写 H。后缀 `.sih.md` 标记司衡治理文档。文件名承载人类浏览意图，id 承载引擎无歧义标识，两者不强制一致。

## 六、字符约束

以下约束应用于正文叙述文本，不修改代码块、Mermaid 块与 frontmatter 内的内容。

### 6.1 字符与标点

- 仅使用 ASCII 字符与 CJK 字符，禁止 emoji 与其他非 ASCII 符号
- em-dash（U+2014）在中文词句间作连接时，替换为全角冒号 `：`
- middle dot（U+00B7）替换为 ASCII 连字符 `-`
- 左右弯引号替换为直引号 `"`
- 右箭头替换为 `->`，左箭头替换为 `<-`
- 不等于号替换为 `!=`

允许的 CJK 标点：U+3001 U+3002 U+FF0C U+FF1A U+FF1B U+FF08 U+FF09 U+300A U+300B U+300C U+300D。

### 6.2 结构约束

- 正文禁止使用水平线 `---`，水平线仅用于 frontmatter 分隔符
- 使用二级标题 `##` 分节，不使用水平线分隔
- 表格不超过 3 列，超宽表拆分为列表或子节
- 所有代码块必须指定语言标签：`mermaid` `text` `yaml` `json` `rust`
- 禁止空代码块，禁止 ASCII 艺术图，流程图使用 Mermaid `flowchart`

### 6.3 排版约束

- 粗体（`**`）仅用于术语定义语句与突出数值，不用于正文或示例
- 列表项保持简洁，每项一个概念，最多 2 级嵌套
- 冗长内容使用段落而非列表
