# R5: 最小可行体系 -- 司衡体系中什么是负载的，什么是装饰的

## 任务

你是一个极简主义架构师。你的唯一任务是：**找出司衡体系中哪些部分是真负载（删除后体系不再成立），哪些是装饰（删除后体系功能不受影响）**。

不审对错。不审哲学。只做一件事：**逐层删除，检验存活**。

## 强制约束

1. **思想实验：逐层删除。** 从最高层开始，假设你删除一个概念/一条道/一个机制，然后判断：剩余的体系是否仍然构成一个可工作的代码工程治理系统？

2. **删除测试按以下顺序执行：**

   | 删除对象 | 测试问题 |
   |---------|---------|
   | 元（四元体系） | 删除元，道/法/术/鉴仍然成立吗？ |
   | 道一 | 删除道一，G1（Scope Boundary）还有依据吗？ |
   | 道二 | 删除道二，G2（Causal Alignment）还有依据吗？ |
   | 道三 | 删除道三，意图恢复的工程流程还有依据吗？ |
   | 道四 | 删除道四，体系的自我修正机制还有依据吗？ |
   | 鉴（九段式） | 删除九段式，道层的主张还能被检验吗？ |
   | F/G/J 力度体系 | 删除力度分级，validator 还能工作吗？ |
   | 三机体系（iCL/iWW/iCT） | 删除三机分权，治理流程还能工作吗？ |
   | 六层脉络（道法术几约形迹） | 删除六层，四层（Observations/Principles/Guidelines/Mechanisms）够用吗？ |
   | 文档生命周期（stage 1/3-3/3） | 删除 stage，治理流程还有意义吗？ |

3. **负载性判定标准：**

   - **真负载**：删除后至少一个工程治理功能不再可操作
   - **条件负载**：在特定条件下负载，在其他条件下不负载
   - **装饰**：删除后工程治理功能不受影响，所有可操作功能可由其他部分独立承担

4. **必须区分两种负载：**

   - **哲学负载**：删除后哲学体系的内部自洽性受损
   - **工程负载**：删除后工程治理可操作性受损

   核心问题：工程负载的部分是否正好是哲学负载的部分？还是两者不同？

5. **最终产出：最小可行体系描述。** 描述"如果只保留真负载部分，司衡的最小可行内核是什么"，用不超过 5 条核心主张概括。

6. **不得使用价值判断。** 某物是装饰不代表它不好，某物是负载不代表它好。你只描述依赖性，不做价值判断。
